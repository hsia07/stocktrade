# stocktrade 自動修碼 / 自動驗收 / 自動 commit-push 治理整合包

這是一套面向 **長時間本地自動修碼**、**嚴格驗收**、**安全 commit/push** 的治理整合包。
它不是單純說明文件，而是把流程、法典、權責、機器可讀 manifest、hook/CI 範本、排程範本、報告模板一起整理好。

## 目標
1. 讓本地 coding agent 可以高度自動化執行。
2. 讓「是否通過」不能由同一個執行體自己宣判。
3. 讓高風險變更必須經過更高等級審核。
4. 讓整套流程可以長時間跑，但仍保留可追責、可回滾、可人工接管。

## 核心原則
- 寫 code 的 agent 不能決定 pass。
- opencode 只負責格式化與橋接，不負責裁決。
- 驗收腳本是第一裁判。
- ChatGPT + opencode 擔任高階審核 / 流程設計 / 規則收斂。
- 你保留最終放行高風險變更的權力。

## 建議閱讀順序
1. `00_START_HERE/01_總覽與導入順序.md`
2. `00_START_HERE/02_7日導入計畫.md`
3. `01_CONSTITUTION/01_全自動修碼與驗收治理憲章_v1.md`
4. `01_CONSTITUTION/02_角色權責與RACI矩陣.md`
5. `07_REPO_SPECIFIC_STOCKTRADE/`

## 注意
本整合包是 **極嚴格治理框架 + 可落地範本**。
你仍需要依 repo 實際結構，逐步把 hook、CI、scheduler、manifest 路徑接上去。

## 最重要的一句話
**全自動可以，但通過不能由同一個執行體自己宣判。**
