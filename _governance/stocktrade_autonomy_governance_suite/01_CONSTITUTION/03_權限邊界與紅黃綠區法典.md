# 權限邊界與紅黃綠區法典

## 綠區：可全自動
允許：
- 自動修改
- 自動驗收
- 自動建立候選 commit
- 自動推工作分支

建議範圍：
- `reports/`
- `manifests/`
- `policy/`
- `scripts/`
- `tests/`
- `.githooks/`
- `.github/workflows/`

## 黃區：可自動修，但不得自動合併
允許：
- 自動修改
- 必須產生 review packet
- 必須通過高階複審

建議範圍：
- `RISK_RULES.md`
- `SOURCE_OF_TRUTH.md`
- 非 live 的設定模板
- 非核心契約文件

## 紅區：禁止無監督自動改動
禁止：
- 直接自動修改並合併
- 自動 push 主分支
- 自動開 live 權限
- 自動繞過人工審核

建議範圍：
- `server_v2.py`
- `index_v2.html`
- broker adapter
- live secrets
- 風控主控
- mode transition / recovery / halt 主控
- reconciliation / execution ledger

## 紅區例外
如需改動，必須：
1. 產生 high risk exception request
2. 通過驗收腳本
3. 經 ChatGPT 高階複審
4. 由你明確放行
