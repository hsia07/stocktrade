# 重試 / 停損 / 升級人工法典

## 重試上限
- 同一輪整體重試上限：5 次
- 同一失敗類型重試上限：3 次
- 同一 forbidden path 命中：1 次即升級人工
- 同一高風險變更：不得自動重試超過 1 次

## 停損條件
符合以下任一者，流程必須停止：
- 驗收腳本判定資料一致性風險
- 命中紅區修改
- commit message 不合法
- 缺證據
- 缺 manifest
- 規則衝突未解

## 升級人工條件
- 連續 3 次無進展
- 同一錯誤重複出現
- 驗收 pass 但高階審查疑似假通過
- 需修改紅區
- 涉及 live / secrets / broker

## 合法等待狀態
當外部依賴阻塞時，允許：
- 補文件
- 補測試
- 補 manifest
- 補 stub / adapter 介面
不得：
- 宣稱通過
- 推進依賴該成果的後續輪次
