# 總覽與導入順序

## 你現在要做的不是「直接開全自動」
而是把全自動拆成 6 層：

1. 規則層：母表、法典、RACI、禁止事項
2. 模板層：opencode 固定輸出模板、review packet、report 模板
3. 執行層：本地 coding agent、Ollama / 本地模型
4. 驗收層：機器可讀 manifest + 驗收腳本
5. 鎖門層：pre-commit / pre-push / CI / branch protection
6. 高階審查層：ChatGPT + opencode + 你

## 最短導入順序
### Day 1
- 先放入本整合包
- 建 `current_round.yaml`
- 建 `agent_policy.yaml`
- 不改 repo 核心，不開 auto push

### Day 2
- 接 `pre-commit`
- 接 `pre-push`
- 驗證 commit message / forbidden paths / evidence 三件事

### Day 3
- 接 GitHub Actions
- 讓 push 到工作分支時自動跑驗收
- 不通過則失敗

### Day 4
- 接 scheduler
- 定時拉 repo / 看新 commit / 跑驗收 / 產出報告
- 仍禁止 agent 自動改紅區

### Day 5
- 接 opencode 固定模板
- 讓每輪指令 / 回報 / 失敗報告格式一致

### Day 6
- 讓 agent 可自動改綠區
- 黃區改動必須產生 review packet
- 紅區仍需高階審核

### Day 7
- 開啟自動建立候選 commit / 候選 PR
- 高風險變更禁止自動 merge
- 啟用人工最終放行機制

## 導入原則
- 先鎖門，再自動化。
- 先機器驗，再高階審。
- 先綠區，再黃區，最後才碰紅區。
