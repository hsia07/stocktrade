# 分工總表

## 你
- 最終放行高風險變更
- 決定是否允許升輪
- 掌握 live 權限
- 決定是否接受重大規則修法

## ChatGPT
- 定義每輪真正驗什麼
- 收斂規則與通過標準
- 抓規則矛盾 / 假通過 / 表面修補
- 判定核心缺口 vs 表面問題
- 擔任高階審核官 / 流程設計官

## opencode
- 把高階規則轉為固定模板
- 生成標準化執行指令
- 維持每輪輸出格式一致
- 擔任格式化指令官 / 執行橋接官

## 本地 coding agent
- 改 code
- 跑測試
- 產出 diff
- 產出候選 commit
- 不得自行宣判通過

## 驗收腳本
- 唯一第一層裁判
- 結構驗收 / 行為驗收 / 流程驗收 / 回歸驗收
- 直接輸出 pass / fail 與失敗分類

## CI / Hook / Scheduler
- 鎖門
- 定時執行
- 阻擋未通過變更
- 管理流程節奏
