# 7 日導入計畫

## Day 1：建立治理骨架
- 放入本整合包
- 建立 `automation/current_round.yaml`
- 建立 `automation/agent_policy.yaml`
- 建立 `reports/`、`manifests/`、`.githooks/`、`.github/workflows/`

交付物：
- `current_round.yaml`
- `agent_policy.yaml`
- `allowed_paths.txt`
- `forbidden_paths.txt`

## Day 2：先把 commit 鎖起來
- 啟用 pre-commit
- 檢查 commit message
- 檢查 forbidden paths
- 檢查 required evidence

交付物：
- `.githooks/pre-commit`
- `.githooks/pre-push`

## Day 3：加 CI Gate
- 建立 GitHub Actions
- push 工作分支自動驗收
- CI 失敗就不允許宣稱通過

交付物：
- `.github/workflows/validate-round.yml`

## Day 4：排程巡檢
- 建立 scheduler
- 定時：
  - `git fetch`
  - 比較新 commit
  - 判斷是否觸發修碼
  - 跑驗收
  - 產出報告

交付物：
- `automation/scheduler/run_cycle.ps1`
- `automation/scheduler/run_cycle.sh`

## Day 5：接 opencode 模板橋
- 讓 ChatGPT 給出的審核要求可轉成固定模板
- 每輪統一輸出格式
- 回報不可飄移

交付物：
- `09_OPENCODE_BRIDGE/01_opencode固定模板.md`
- `03_MANIFESTS/templates/opencode_instruction_template.md`

## Day 6：打開綠區自動修碼
- 只允許 agent 修改綠區
- 黃區需 review packet
- 紅區一律禁止直接自動變更

交付物：
- `agent_policy.yaml`
- `review_packet.example.md`

## Day 7：開候選 commit / PR
- 通過驗收才允許建立候選 commit
- 高風險只允許候選 PR，不可自動 merge
- 開啟升級人工流程

交付物：
- `escalation_report.md`
- `high_risk_exception_request.md`
