=== PROBLEM STATEMENT ===

After BLOCKER_I (dispatch scaffold), the system has the ability to
determine next round and create scaffold, but there is no entry point
for Telegram/API/cron to trigger the dispatch. A handler is needed
that wraps the full gating chain and only dispatches when safe.

=== SOLUTION ===

- auto_advance_handler.py: handle_auto_advance_trigger()
- Accepts round_result, checks can_dispatch_next_round (which internally
  gates on auto-candidate status, signoff, blockers, evidence, pause,
  sequence end)
- If dispatchable, calls dispatch_next_round to create scaffold
- Always returns merge/push/promote/pr_created = False
- Bounded auto-mode and manual-review path preserved via delegation to
  can_auto_advance / can_dispatch_next_round
