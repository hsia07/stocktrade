Background

R021 topic from v2 law: 重要設定鎖定

Implementation
-------------
Created settings_lock.py with:

1. SettingsLock class:
   - Protects important settings from unauthorized modification
   - lock_setting(): Lock/unlock specific settings
   - is_locked(): Check if setting is locked
   - request_unlock(): Request authorization token to unlock
   - confirm_unlock(): Confirm token to unlock a setting
   - validate_modification(): Check if modification is allowed

2. Default protected settings:
   - chain_status
   - current_round
   - last_completed_round
   - auto_run
   - chain_control.status
   - phase_state

3. Audit trail:
   - All modification attempts logged to settings_lock_audit.jsonl
