=== PROBLEM STATEMENT ===

After BLOCKER_E (automated signoff), the system can determine that
auto-advance IS allowed, but there is no mechanism to dispatch the
next round. A human must manually create the next candidate branch.

=== SOLUTION ===

- ROUND_SEQUENCE: explicit ordered list of blocker IDs
- determine_next_round(current_round_id): returns next round from sequence
- can_dispatch_next_round(round_result): checks auto-clear + next exists
- dispatch_next_round(round_result): creates candidate scaffold directory
  with evidence.json and task.txt skeleton
- All dispatch results explicitly set merge/push/promote/pr_created = False
- Bounded auto-mode and manual-review path preserved (delegates to
  can_auto_advance which already enforces those)
