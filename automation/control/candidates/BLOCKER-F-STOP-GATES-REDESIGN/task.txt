=== PROBLEM STATEMENT ===

can_auto_advance() had no distinction between auto-candidate rounds and
bounded auto-mode rounds. All rounds were subject to the same STOP_GATES
(merge/push/activation/authorization), making auto-advance impossible.

=== SOLUTION ===

Redesigned can_auto_advance() with two paths:

Auto-candidate path (formal_status_code == auto_candidate_ready):
- Bypasses manual STOP_GATES (merge/push/activation/authorization)
- Still blocked by safety gates: blockers, evidence, pause
- Returns "auto_candidate_path_clear" on success

Bounded auto-mode path (all other status codes):
- Original behavior fully preserved
- All STOP_GATES enforced
- candidate_ready_awaiting_manual_review still blocked by review_gate
- Returns "all_conditions_met" on success

Runtime behavioral tests covering:
