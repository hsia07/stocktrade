SIDELINE-TELEGRAM-PROGRESS-EVENT-LABEL-MAPPING-FIX

## Task Description
Fix event label mapping in Telegram Progress Notifier/Sidecar to ensure:
- START events display as START (not PROGRESS)
- PROGRESS events display as PROGRESS
- COMPLETE events display as COMPLETE
- ERROR events display as ERROR

## Problem
The sidecar was mapping all non-completed/failed/paused/stopped statuses to `send_progress()`, 
causing START events to be incorrectly displayed as in progress.

## Solution
Modified `telegram_progress_sidecar.py` to check `formal_status_code` for START keywords 
(start/begin/init/commence) and explicitly call `send_start()` instead of `send_progress()`.
