# 05_每輪詳細主題補充法典_機器可執行補充版

## 文件性質
- 本檔是 **逐輪詳細主題補充**，目的在於把原本偏籠統的輪次說明補到可供自動流程與審核使用的粒度。
- 本檔 **不得推翻或放寬** 既有正式法典。若與 02 / 03 / 04 / 01 衝突，一律以前者為準。
- 已有明確主題名稱的輪次，沿用既有名稱。尚未正式命名的輪次，採 **占位主題名稱**，供之後逐輪落名。
- 占位主題名稱的作用是 **縮限範圍**，不是授權自由發揮。

## 固定解讀規則
1. 每輪只允許 **一個唯一主題**。
2. 每輪預設只允許 **1 個主檔 + 最多 2 個高度相關中小檔**。
3. 每輪都必須先整輪重檢，再補缺口，再驗證，再收斂成獨立 candidate。
4. 每輪都必須具備：獨立 candidate branch、獨立 evidence package、獨立 RETURN_TO_CHATGPT。
5. 每輪若未達 `candidate_ready_awaiting_manual_review`，**不得進下一輪**。
6. 不得把純側線 / 純工具 / 純啟動器修補視為本輪核心完成。
7. 不得自動 merge，不得自動 push master，不得把後一輪 candidate 視為前一輪正式收口。
8. 到 merge 決策點時，必須等待使用者明確說「好 / 同意」後，才可提供 merge 指令。

## 各輪共同最小完成標準
每輪都必須同時滿足以下條件，才可進入下一輪：
1. 該輪唯一主題已完成實作。
2. 該輪可重跑測試 / 驗證通過。
3. 該輪 evidence package 完整。
4. `validate_evidence.ps1` 真正執行且 PASS。
5. `formal status code = candidate_ready_awaiting_manual_review`。
6. candidate branch 與 candidate commit 可審計。
7. 不存在 fabricated evidence。
8. 不存在越權修改。
9. RETURN_TO_CHATGPT 完整有效。

## 固定禁止修改範圍
- master
- `server_v2.py`（除非第 6 輪核心交付證明必須，且需先 blocked 申請授權）
- broker / live / 交易 / 風控主流程之授權外核心檔
- 任何當前 phase 以後主題內容
- promotion / panel / governance 若與該輪核心主題無直接關聯，不准順手修改
